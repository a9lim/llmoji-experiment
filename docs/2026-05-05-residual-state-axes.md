- Deployment surfaces (drive the chart marker shape, post-2026-05-06): **Claude Code (only)** = **124**; **any claude.ai** (incl. faces also in Code) = **65**; **neither / HF corpus only** = **6**. Source: a9's local `~/.claude/kaomoji-journal.jsonl` + the claude.ai exports listed in `llmoji_study.local_emissions.DEFAULT_CLAUDE_EXPORTS`.
- Display: **color = proportional RGB-blend of BoL shares** for every face (uniformly, even where GT is available — surfaces the synthesis-vs-emission gap rather than collapsing shared faces to GT). Each face's marker is a per-quadrant-weighted mix of `QUADRANT_COLORS`; ties no longer hide behind the QUADRANTS-order argmax. **Marker shape = deployment surface**: ○ = Claude Code only, ◇ = any claude.ai, □ = neither. Hover surfaces the GT-overlap category, top-2 BoL quadrant components, and emit weight. Mirrors the per-face PCA coloring convention from `scripts/local/97_build_per_face_pca_3d.py`. (Companion view: re-run with `--color-by gt` to blend Claude-GT shares instead, with non-GT faces black.)
- Per-face soft distributions are exported alongside the modal labels in `data/harness/wild_faces_labeled.tsv` as `bol_share_<Q>` and `gt_share_<Q>` columns (one each per quadrant in ['HP-D', 'HP-S', 'LP', 'NP', 'HN-D', 'HN-S', 'LN', 'NB', 'HB']); these are the distribution-vs-distribution-honest read on every face, unlike `bol_quadrant` / `gt_quadrant` which are the modal argmax collapses (HP wins ties because of QUADRANTS index order).
